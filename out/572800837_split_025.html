Extremely Basic Editing and Creation of text files
 
It is recommended that you experiment with a simple text editor like gedit and familiarise yourself with its basic functionality for the creation and editing of text files. It will be important to learn this in order to both create your own text files, and for the customization of your working environment and amending configuration files.
 
Input and Output Streams
 
There are three streams
     stdin – Standard input
     stdout – Standard Output
 
stderr – Standard Error
 
     The file descriptors for these are respectively 0, 1, 2 which will be used in the redirection of I/O streams.
     stdin – 0
     stdout – 1
 
stderr – 2
 
Input and Output Redirection
 
It is important to understand the different input and output streams in Linux, and the core concepts of redirection and piping.
 
Piping of Input/Output Streams
 
An example of piping that draws on commands that we shall meet soon is
     $ ps ax | grep help
     This command line pipes the output of the ps ax command, which lists all of the running processes on the system, through to the input of the grep command, which allows us to search that input for, in this instance, the string
 
Redirection of Input/Output Streams
 
There are a number of ways of redirecting the different input/output streams, which can then be combined with other programs to perform different functions.
     We make use of the > and < operators in our redirection. When only one is used it will cause the result to be overwritten, when two are use it will append any results.
     Overwriting
 
> - standard output
< - standard input
2> - standard error
 
     Appending
 
>> - standard output
<< - standard input
2>> - standard error
 
     A number of examples will be used to show the different behaviours..
 
Redirect the output of a program to a file
 
$ ls –l > ls.txt
 
Redirect the stderr from a program to a file
 
$ ls –la 2> error.txt
 
Redirect stdin to a program
 
$ wc < cat_result.txt
 
Redirect stderr to stdout
 
$ ls –l 2>&1
 
Redirect both stdout and stderr to a file called output.txt
 
$ ls –l > output.txt 2>&1
     These are just a few examples of redirecting input/output streams, but are indicative of the degree of control and power that is achievable.
 
cat
 
cat can be used to redirect stdin into a file by using cat with no parameters other than a file name
     $ cat > cat_stdin.txt
     This allows us to create a text file line by line, using ctrl-D (control-D) to indicate the end of file, EOF.
     In its basic form for concatenating files cat takes two or more filenames and adds the contents of one file to that of the previous. Since we can use cat to view the contents of a file we shall look at two files and then concatenate them, redirecting the output to a file called cat_result.txt, and look at the result in a screenshot.
     $ cat cat_ex1.txt cat_ex2.txt > cat_result.txt
     We can also use cat to take input from stdin and redirect it to a file. We use the following command
     $ cat > filename
     then we can add lines of text until we are finished. Press ctrl-D (control-D) to end the input.
     If the file does not exist it will be created. If it does exist then it will overwrite any contents already there.
     To avoid this and add text to the end of the file we can use the command line
     $ cat >> filename
     This will concatenate our new input to the end of the existing input in the file.
 
echo
 
We can use echo to redirect input to a file thus
     $ echo “A string of text” > filename
 
tee
 
The tee command allows one to read the standard input and write it to one or more files, duplicating its output. It is widely used in conjunction with pipes and filters. A simple example that takes the output of the ps which lists running processes, and pipes it into both a new text file and at the same time into the less command
     $ ps ax | tee listing.txt | less
     Thus ends our brief look at input/output streams, redirection of I/O streams and the creation of very basic text files. We shall now take a look at how we find programs, files, etc. and search text files for information.
