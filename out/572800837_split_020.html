Discerning Commands
 
By this stage, you should have some basic understanding of command lines and how they can amplify your Linux experience. You have been able to practice some simple command lines, navigate through information and find applications which you can use to customize your operating system. You have a good amount of knowledge in Linux, though here is still some information that you need. You need to properly discern what a command is.
     Commands are at the heart of any Linux system. There are four different types of commands. These are explained as follows: -
 
Executable programs. These are the programs that you have experienced when navigating files, for example the /usr/bin. Here is where you are likely to find programs that have been written in C and C++. These are known as compiled binaries. There are also programs which use scripting languages, such as Python and Ruby
A Shell Function. You have learned how to execute a shell prompt. Shell functions are miniature shell scripts which are incorporated into a particular environment.
A Command built into the shell. cd is an example is a shell bulletin. Bash is able to provide internal commands which are referred to as shell bulletins.
An Alias. These are commands that are built up from other commands, and you can use these to define yourself.
 
     Here are some basic commands that you can use to help you improve your Linux experience.
 
which – this will help you to locate a command
help – this will display a reference page for the shell bulletin
type – this will display information about a command type
man – this will display an on-line command reference
 
     Using which
     Within an executable program that has been installed within a system, it is quite possible to find that more than one version exists. This possibility increases significantly when a large server is being used. Therefore, if you would like to find the exact location of an executable program, you should use the which command. This command does not work for builtins or aliases as these are substitutes for executable programs. Type it in as follows.
     [name@mylinux me]$whichls
     /bin/ls
     Using help
     When you are using bash, there is a help facility that has been built-in especially for shell bulletins. All you need to do to access it is type “help” followed by the name of the shell bulleting. Additionally, you could use the –m option as well, and this will alter the format of the output.
