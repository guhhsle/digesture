The Installation of Linux
 
There are a few different ways you can install the Linux operating system onto your technological device, but the most popular way is by downloading the program and burning a live CD version. Unless you are using some sort of netbook, this is probably the easiest way for you to get Linux onto your computer or desktop personal computer.
     What burning a live CD requires is a computer that has disk-burning capabilities and that can connect to the internet. You’ll need to find a source online that has the version of Linux you have chosen, or you can download one of the more popular versions directly from the main Linux website.
     Some of the versions you’ll find have both a 32-bit and 64-bit version available for each desktop environment. If you are in doubt, stick with the 32-bit version.
     For those who are curious about the difference, here is the main one: the 64-bit versions have been optimized for better interaction and desktop environments. Think of it as an upgrade of sorts. With 32-bit operating systems, you are limited to 4096 MB of RAM (the overall memory capacity of your operating system). With this 32-bit system, all of the processes are given 4 GB of virtual memory to play with, which is then split into 2 GB of user space the application can utilize at a time. What all of this means is, even though your computer might allow for more RAM space, the 32-bit version of any Linux operating system is only going to be able to play around with so much memory.
     So, if you are installing a 32-bit version of Linux onto a technological device that has 10 GB RAM, you are still only going to be able to utilize 4096 MB of it because that’s all the designed operating system can handle at once.
     Not only does the 32-bit version of Linux operating systems limit the overall amount of memory it can use and address at any one time, but its other devices also take up room in that same space. The video card, the motherboard, and everything else like that within the realm of your computer steals even more of that 4096 MB space, meaning the original operating system has even less to work with.
     Think of it this way: you are standing in a 10-acre field. That field is the overall memory of your technological device. Now, you want to build a barn for everything to function in, so you build a board that’s 4 acres in diameter. That barn is your entire operating system: it’s 4 acres, but it can’t utilize any of the other 6 acres that outside the walls of the barn.
     Then, in come the animals, the feed, the tackle equipment, the office space, the whiteboards to keep the animals on schedule, and everything else the barn needs to run efficiently. The animals and the feed are like the video card and the motherboard—they fit inside of that barn (or the operating system), but they are only allowed to function within that barn. This means that the more things you add to the barn (or your operating system), the less space you have to move around in that barn (as in, your “walking space.”)
     Overstuff your barn, and you can’t walk around in it any longer, even though you have 6 acres you can expand into.
     Make a little more sense?
     With the 64-bit versions of the Linux operating system, it can access 17.2 billion GB of system memory at a time. This effectively banishes all limits of computer access far into the future. This also means your video card, motherboard, and all other peripherally-working systems within your computer will no longer be taking up the much needed space within the memory of the operating system. Because of this massive upgrade, more people have switched to using 64-bit versions of all operating systems on the open market, and this has led to all of your favorite applications and programs being revamped so they work flawlessly on this type of operating system version.
     The reason why we say to default to the 32-bit version if you aren’t sure is because these versions are perfect for low-level applications and programs. If you are a true beginner, this will allow you to tinker with the basics before getting ahead of yourself and clogging your computer up with programs you don’t know how to navigate yet.
     A 32-bit version essentially puts a temporary muzzle on you and forces you to get to intimately know Linux and your computer before opening up the floodgates and submerging yourself entirely. However, if you want the best right out of the gate, go with the 64-bit version and don’t look back!
     Now, once you have the version you want, download the main file. This is usually an ISO file that will download itself onto the computer you are using (and that has the ability to burn disks). After the initial file is done downloading (and it might take a while, so go get some coffee), open up the burning program you are going to use and burn that program onto a blank CD.
     Then, after it’s done burning (and please make sure it’s before you do this), eject the CD and put it into the computer you want to install Linux onto. Make sure your computer is off when you insert the CD, but if your computer is on when you do, just restart it. When your computer begins to boot itself up (or reboot itself, if you had to), there will be a sign prompting you to “press any key to boot from CD.” Press a key and let the CD boot up your computer. After it does, it will give you a prompt screen that will ask whether you want to install Linux or try it out. Click the option to install it and wait for the launcher to pop up. Then, simply follow the command prompts in order to download it to your computer!
     Now, this is already assuming you know you want this desktop environment. If you are still trying things out, simply click the option to try out the Linux program and it will load the operating system without actually installing it!
     However, if you have chosen to download it, you can launch the installer right from your desktop. Once you have tried it out and found that you do wish to use this desktop environment, there should be a desktop icon that denotes the fact that a burned CD is within the computer. Right-click it and choose “download,” and follow the command prompts that pop up after that!
     There is another way to download Linux that’s very easy, and this is via the live USB method. This way will also require a computer that’s not the computer you are loading Linux onto, so make sure you have one available that has at least one USB port. To begin, simply go to the main Linux website and find the version of the operating system you want. Download the .ISO file onto the computer, then open up the USB thumb drive on your computer.
     If you don’t know how to open up a thumb drive on your computer, it’s usually pretty simple: if the window for the thumb drive doesn’t open up after automatically inserting it, then find the “file manager” program and open it. A window should pop up, and to the left of the window should be a listing of all the main files and folders for your computer. One should be labeled “thumb drive” or “USB,” and that’s the one you want to open.
     Once the file is done downloading, click and drag the file from your computer desktop to the “thumb drive” or “USB” file you have opened in that previous window. Give it some time to transfer the file from the desktop to the thumb drive, and when it’s done, eject the thumb drive from the computer and go over to the technological device you want to install Linux onto.
     Now, this way will take a couple more steps, but we will walk you through it. Insert your USB thumb drive into the computer in question and turn on (or restart) your computer. But, as soon as the computer restarts or begins booting up, hold down the “Delete” button. This will start your computer up in something called a BIOS setup. Once the bootloader screen pops up, there will be several prompts on it asking you what you want to do. Click on the “Boot” section of the BIOS setup and look for the section that says something to the effect of “change PC boot order.”
     Once you choose it, you should have several different things that are set in a specific order. This boot order means that, upon booting up your computer, these ports are the first to register. Your CD drive should be near the top, which is why using the live CD doesn’t require these steps. Most computers are setup to quickly read what’s in the CD drive before it does anything else.
     Find the option that says “USB hard drive” and move that to the top of the list. Save the settings, exit the BIOS, and then restart your computer, all while the thumb drive is still inserted. Once your computer reboots itself, it should take you to the main menu, where you can now boot into and choose to install the Linux operating system you have chosen!
     Keep in mind, each distro (or distribution of a different Linux operating system desktop environment) will have slightly different needs once the installation process begins, but this is how you begin this process. All of the screens will prompt you and tell you what you need to do in order to continue the installation, so take a deep breath and try not to feel too overwhelmed.
     Once you have Linux installed on your computer, it’s time to delve into the main purpose many people choose Linux for in the first place: coding and programming.
