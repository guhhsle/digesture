Programming in Linux using C++
 
So now that we’ve gotten all of that done, it’s time that we start talking about building actual programs in Linux. The first language that we’re going to be working with is C++. A huge amount of software for Linux is written in C++, and it’s just a monolith of a language that’s majorly stood the test of time. Even today, when one needs an efficient language that can get things done cleanly, C++ is the go-to language. This is why it’s the industry standard for resource-intensive applications which don’t need a rapid prototyping phase.
     There is a multitude of different reasons to use C++. However, in the age of rapid prototyping, quick returns, and lots of computing power, there are in many cases better choices. With that said, for serious enterprise development projects, C++ remains one of many go-to. Beyond that, there’s one very simple reason to learn C++: no language will get you more intimately connected with your computer while still offering you the complexity of object-oriented programming like C++ will. In other words, C++ is an amazing language to because it gives you a huge amount of programming utility, but it also offers you the ability to really know and learn what you’re working with.
     Perhaps, in some ways, it would be better to start with Python first; after all, Python is in many ways a far easier language to pick up and learn than C++ is. However, I’m choosing to put C++ first because it’s better to start off hard and learn the concepts. Python arguably offers too much abstraction for a new programmer, who is benefited the most from learning properly everything they’re working with.
     Programming in C++ in Linux is extremely easy. All C++ programs are compiled. This means, naturally, that you need a compiler. You’ll have to first install this. Open up a Terminal window, then type the following:
     sudo apt-get install gcc gcc-c++
     This may vary depending what distribution you’re using. For example, on Fedora, the command would be sudo We already talked about these sorts of differences in the chapter prior, but it’s worth rehashing them to ensure that you’re ready to deal with them.
     After your compiler is installed, you just need to learn how to compile. All you do is navigate to the location of your given .cpp file. Then, you type in the following:
     g++ -o executableName sourceName.cpp
     Then you just type executableName into the command prompt to run the file.
     So now let’s get to creating our first program. What you’re going to do is go into your text editor of choice and then start writing code. Save your file first thing so that you get syntax highlighting. Save it wherever you want and under whatever name you want, remembering that the filename extension must be
     Within your text file, type the following:
     #include
     using namespace std;
     int main() {
     cout << “hello world\n”;
     return 0;
     }
     From here, you need to save it and compile it. It should print “hello world” out to the console. If it does, then congrats! You just wrote your first C++ program. How does it feel?
     Don’t worry though; there’s a long path ahead. Let’s start by breaking this program down line by line. If something is a little more advanced than you’re ready for at this level, I’ll just give a cursory explanation before we move onto the next topic.
     #include
     This is a preprocessor In other words, what this does is tell the compiler things it needs to know about the code before it does absolutely anything else with the program. In this specific instance, it’s telling the compiler to include a certain Libraries (also called headers, depending on what language you’re working with) are chunks of pre-defined code that exist in order to, essentially, make your life easier. The iostream library is the specific C++ library tailored towards defining input and output streams, our primary example here being the console. Thanks to this line, we’re able to output text at all.
     You can create your own libraries and headers, but that’s something far beyond the scope of what you want to do right now. Knowing this is an important part of modular programming, though, so bear it in mind; every program can be broken down into multiple programs and multiple chunks of code.
     using namespace std;
     Technically, this line isn’t necessary, but we’re using it anyway. This tells the compiler that we’re going to be using a certain namespace within our code. Namespaces are functionally similar to headers, in some ways; for example, they allow you to predefine chunks of code and necessary functions, then utilize them in other code files. Headers, in fact, are generally composed of different namespaces and preprocessor directives. By declaring this, we are able to quickly and efficiently output text. We’ll get to that in just a second though.
     int main() {
     // ...
     }
     This is just a function declaration. This is, in fact, the entry point of the code. All programs must have an entry point; it’s where the compiler or interpreter will start to read code. Some programming languages use implied entry points at the first instruction, and some choose to require an explicit entry point. We’ll be working with both sorts in this book. C++ is the latter. We’ll be talking about function declarations more in depth soon, but for now, just remember that every program must have this.
     cout << “hello world\n”;
     There’s a lot going on in this line, so we’re going to go ahead and break it down one component at a time. 
     First, we have This just stands for console This is the main method by which you can print out characters to the console in C++. This is actually a member of the std namespace. If we hadn’t used using namespace we’d have to call this every time by saying std::cout instead of just This can make quite a big difference in terms of efficiency of typing. The argument against this is that including an entire namespace is a bad habit and can lead to inefficient code. This is your discretion. Eventually, you’ll come to know whether or not to do it just based off of the scope of the project and the parameters thereof.
     Then, we have This is, for lack of a better explanation at this point, the output operator. Whenever you’re sending something to be output, you need to use this. You can also break up text output into many different segments using this. If you need to print out variables, you’ll do so by dividing output strings and variables. You’ll see that in a second.
     The last major part of the line that we need to talk about is the This is called an escape sequence. Escape sequences are used in order to denote things to the compiler that are difficult to intimate otherwise. For example, whitespace is a non-factor for C++ programs - if it weren’t for coding conventions, you could space out your code however you wanted, and as long as all of the syntax was right, it would still compile. Because of this, there is no real way to put a tab at the start of a line, nor to initiate a line break within a string. With escape sequences, however, you can do this. In order to insert a new line, you just use \n. In order to denote a tab at the start of a line, you use /t. This is the general convention. There are many other escape sequences, all of which with their own unique use cases. However, these are the two most prevalent sorts that you’re going to run into.
     Also, note that the line ends with a semicolon. In C++ (and most C-style languages), every raw code statement ends with a semicolon. This is important because it means the termination of a statement, which lets a compiler know when a “sentence” of code is complete and it can move on to the next part.
     The last line is just this:
     return 0;
     All this indicates is that the program is ending. What it’s really saying is telling the compiler, “hey, we’re in the main function of the program, and I’m returning a value, so now the main function is closing, and the program can be closed.” Since the main function of a C++ program is an int, it must return a number.
     Values and Variables in C++
     Values and variables in C++ are really complex beasts, yet they’re also really simple. They’re complex because you have to know exactly what you’re wanting, but at the same time, they’re simple, because they can only take so many forms, and they’re far more finite than variables in, say, Python.
     There’s not even an inkling of complexity to them once you start to have a firm grasp on the concepts you’re working with, but at first, things can be a bit confusing. My intent here is to make them as not confusing as possible.
     All of the following are valid values in terms of computer science:
     14670
     .000013
     13.46
     “I would like to buy a new car.”
     b
     True
     So how does one actually manage these? How can you “keep” one of these and change it at a later point? Through establishing variables, of course.
     The general way of establishing variables in C++ is like so:
     type variableName =
     So how do you know the type? By looking at the value.
     The essential file structure is going to be much the same; many of your early C++ files are going to share that same file structure, actually, focused around utilizing the console as a means of output. So go ahead and set up an empty shell. From now on, when I say to construct an empty shell in C++, what I mean is to create a file which essentially looks like this:
     #include
     using namespace std;
     int main() {
     // code goes here
     return 0;
     }
     Code operations will be performed within the main function unless otherwise intimated in the chapter, which will be happening when we start looking at C++ functions and function declarations. Anyhow, for right now, we’re just going to be looking at the main function, so it doesn’t matter.
     So, moving from here, let’s look back at those values we established a minute ago and their types. All of them are primitive types except for the string. What this means is that all of them can be created without any extra libraries or class establishments. 
     Let’s say that we wanted to create a variable in our empty shell called which held the value 3. How do you suppose we would do that?
     Well, first, we have to figure out what type it is. Since it’s a number, it’s pretty clear what type it is - it’s an integer! Remember, integers are always whole numbers. 
     Then we have to have a name for the variable in question. We’ve already decided that the name would be bananas, and the value would be 3. So how would we declare this variable? Just like this:
     int main() {
     int bananas = 3;
     return 0;
     }
     This is the general scale for creating variables in C++.
     So what if you wanted to create a string? Well, you’d have to include another header.
     #include
     Then, you could create strings like any other variable as well as access string class member functions which would allow you to do various operations on the strings.
     In order to verify that this is the correct value, on variable bananas, we could actually test it out by adding a console output statement:
     int main() {
     int bananas = 3;
     cout << “ I have “ << bananas << “ bananas!\n”;
     return 0;
     }
     If you were to save and compile this, it would print out:
     I have 3 bananas!
     So there you go! Those are the fundamentals of creating and changing variables in C++.
     Data Sets
     We’re not going to cover the STL in C++ at this point, it goes a fair bit beyond the scope of this book. Because we aren’t going to cover the STL, we aren’t going to be talking about vectors and how you use them in C++. If you’re interested in learning how C++ vectors and sets work, then I’d advise you to Google such; in many C++ programs, given the reasons for using C++ in the first place, it’s often counterintuitive to attempt to use vectors for these sorts of things because they take up an inordinate amount of memory space compared to allocated memory sets. However, they can also be a bit easier to work with than arrays. It can very much be situational, but the chances are that if you’re creating a program where it makes sense to use vectors, it’s probably the sort of program where it makes more sense to use another language entirely.
     Instead of vectors in this section, we’re going to be covering arrays. Arrays are the primary method of storing sets of data in C++. They are highly extensible and can be used in order for storing numerous pieces of related data next to one another. We’ll be talking a bit more about the actual concept here momentarily, but the key thing to know about arrays is that they actually allocate memory space - just enough for all elements listed in the array. The array is then divided into individual spaces, which are the elements that I’m referring to. These elements start counting from zero.
