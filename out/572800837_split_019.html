Using the File Editing Commands and Wildcards
 
This chapter will take a closer look at how these commands work, and how you can use them with wildcards.
     mkdir
     The mkdir command is appliedwhen you want togenerate new directories, and is probably the simplest one on the list. To use it, all you have to do is input the following:
     [me@mylinux me]$ mkdir directory...
     rm/rmdir
     The command is used to erase files and directories from your computer. There are many diverse ways to use the rm command, including:
 
rm file... -  removes files
rm –r directory – deletes directories
rm fileAfileB – deletes both fileA and fileB
rm -1 fileA fileB – prompts the user before each file is removed from the system
rm –r dirA dirB – removes the directories dirA and and deletes all the data within them as well.
 
     As this is Linux, there is no undelete command, therefore, if you use the rm command, ensure that you really want to delete the object before you do because there is no recovery should you make a mistake. Many amateur and professional Linux users have caused great damage to their systems by deleting the wrong files, especially when they combine the rm command with wildcards.
     The easiest way to ensure that you are not deleting the wrong thing, particularly when you are using wildcards, is to use the ls command first. This will permit you to see what outcome your wildcards will have when you employ them with the rm command.
     Should you need to make any changes, you can just go back to the original command using the up-arrow key and begin again until you get the desired result, then you can replace the ls command with rm.
     cp
     In its mostplainform, the cp program moves one file from one directory to another, for example:
     [me@mylinux me]$ cp fileAfileB
     In this case, the contents of fileA are copied into fileB. It is important to remember that should fileB not exist, Linux will create it automatically. However, should it already exist in that location then the file will be overwritten immediately, and the original fileB will be lost forever. To prevent this, a similar cp command can be utilized, with one major difference, as shown below:
     [me@mylinux me]$ cp –i fileAfileB
     In this case, thanks to the –i (interactive) option, if fileB exists in that location then you will be prompted on what action should be taken before the file is overwritten. If you want to ensure that you do not accidentally overwrite anything important when you want to copy a file, use this command instead of the standard one.
     cp can also copy many different files at the same time using this command line
     [me@mylinux me]$ cp file... directory
     In thissituation, the ... indicatesthat the command can be carried out as many times as you wish.
     There are some additional ways you can employ the cp program such as:
 
cp fileA dir1 - This command copies whatever was in fileA into a file called fileA in the selected directory, in this case dir1
cp –R dir1 dir2 - copies whatever was in dir1 to As in the original case, if dir2 does not exist, Linux will generate it for you. In contrast, if the directory DOES exist it will not be overwritten as in the first example. Instead, Linux will create a directory called dir1.
 
     mv
     The mv command is utilized when transferring files from and is also utilized when renaming particular files. For instance, if you would like to transporta particular file from one directory to another, you could enter the following:
     [me@mylinux me]$ mv file... directory
     But, if you wanted to rename a particular file, you would enter this:
     [me@mylinux me]$ mv filenameA filenameB
     It is important to note that just as with the cp command above, when you are renaming should filenameB already exist, then it will be overwritten and replaced with the contents of However, should it not exist, Linux will create the file for you.
     To avoid overwriting any files, it is important to attach the interactive (-i) option to the command so that you can receive a prompt before any changes are made.
     The mv command is useful for many other things. These include:
 
mv fileA fileB fileC dirZ – this command will move and to However, if the directory does not exist, the mv command will fail and you will receive an error message.
mv dir3 dir8 – this will move dir3 to within dir8. However, should dir8 not exist, dir3 will be renamed
 
     Putting Them Together
     Using these commands with wildcards is very simple, and is actually recommended as the commands listed above allow for multiple search results and therefore the results may need to be filtered. Below are a few illustrationsof how to use wildcards with these commands.
 
rm *~ - This will remove ALL files in that directory that end with the ~ character. There are quite a few programs that create backup files using this symbol, therefore running this command every once in a while should help delete redundant files.
mv my_dir ../*.bak command will ensure that all the files labeled in the my_dir subdirectory, and its parent directory are moved to the new folder named my_new_dir.
