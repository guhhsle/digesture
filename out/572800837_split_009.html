The Basic Components of Linux
 
We have already addressed some of the components of Linux, such as the kernel and the shell (or how the user of the Linux system interacts with the software), but there are a few things we have not yet addressed.
     The Linux system library is a set of special programs that are used to access the operating system’s kernel features. These libraries aid in the implementation of some of the most basic functionalities the operating system executes, but you don’t have to do anything in order to get these to work. Just like you don’t have to think about breathing or blinking, you don’t have to toggle anything to tell the Linux system to do these things. Breathing and blinking simply happen involuntarily.
     However, being aware of them will help you to picture how the entire Linux system is set up.
     You also have the system utilities. These are the programs that are responsible for the specialized and individual level tasks. This is the type of software that helps maintain the entire infrastructure of the computer as well as its operating system.
     In other words, this is what keeps Linux as stable as it is.
     Within Linux, there are two different modes: user mode and kernel mode. If you are executing any sort of code or programming language that doesn’t require opening the kernel, then you’re in user mode. Using this mode means you have no access to the basic hardware of the system, so whatever you decide to execute doesn’t affect the low-level tasks stabilized by the kernel.
     It’s the equivalent of allowing a toddler to get inside a cardboard box and color on the cardboard walls. Yes, the color is getting everywhere, but only inside that box. There is no color changing the main walls of your home, forever altering what it looks like until you can get a bucket of soapy water and scrub it down.
     However, kernel mode gives you full access to every single resource the computer has to offer. This is a privileged mode that has the capability of allowing you to alter Linux at its base level. Using the metaphor above, this would be the equivalent of allowing a toddler to not only color on the walls, but take a tire iron to the plaster and pull the electrical wirings through the hole.
     You can completely alter how Linux operates and how it interfaces with the user (a fancy way of saying “how it looks when you open the main screen after logging in”) in this mode if you want. However, as a beginner? I don’t recommend it.
     Linux also has some basic features that come with it that not every operating system has. It’s portable, meaning you can transfer the type of operating system you use to any electronic device you wish, so long as it’s in a form that can be transferred (think a thumb drive or a burned disk). It’s also a multi-user operating system, which means multiple people can access very sensitive information about Linux (like the memory and the RAM) on one system.
     It would be like a split-screen television, where one game player is off shooting a monster, and the other player has fallen off a cliff and landed in the water. Each person can do their own thing within this sprawling world without it affecting the other.
     Linux also has the capability of being a multiprogramming system. This simply means you can run multiple applications at the same time without the system bogging down.
     It also has a hierarchical file system, making things very easy to find once you learn the layout of your operating system, and it also has already-implemented security protocols, such as password protection of specific files and encrypted data. While it’s a multi-user system, the owner can lock it down, if they wish.
     The greatest thing about the Linux operating system, however, is the open world it provides for people who want to learn how to code. You have the capability of accessing every single sensitive file of information, no matter where it’s located, and because it’s a free and open-source licensed operating system, you can tinker as you please without making someone upset with your tinkering. With other operating systems, tinkering usually avoids some sort of warranty. Have you ever heard the term “jailbreaking”? This is a term used when someone bypasses the limitations set on an operating system in order to access those base files to change the way they can interface (or interact) with it.
     There is no jailbreaking or voided warranty when you tinker around with Linux, and this is the main reason why this operating system is the most popular when it comes to those who program, those who want to learn to program, and those who want to use their programming knowledge to take on DIY projects for various purposes.
     Linux also supports most of the programming languages available. Python, Ruby, Vala, C++, Java, Perl, and HTML are just a few of the many that Linux not only understands how to execute, but also understands how to teach. The great thing about this operating system is that, within its own application store, you can download tutorials that run in the background that help you correct the source code you might be tinkering with.
     Did I lose you there? Let’s try this: when you open that little black box you see all those computer whizzes in television shows typing in (called a terminal window), you can begin to type in little lines of code. Those are called “command lines,” and they have to be typed in a certain way with certain characters in order to work.
     If any character is out of place or missing, the line itself won’t work.
     There are programs you can run in the background on the operating system that will help you catch those errors. Some applications will automatically correct the error if it’s something simple, and some will step in and stop you from progressing to the next chunk of code until you have found the error and corrected it. There are a few that go a little more in-depth for beginners, such as flashing possible errors at them on the screen, and some applications can be opened and played like games to help beginners learn the basics before opening their own terminal windows and experimenting.
     All of this is possible on a Linux operating system, which makes it the most sought-after system when someone expresses an interest in learning how to code.
     Another great thing about Linux is how easy it’s to install. However, once the basic operating system is installed, there are specific steps that have to be taken in order to update, upgrade, and install certain packages in order to secure your operating system.
     But don’t worry.
