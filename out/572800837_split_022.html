The I/O Redirection
 
There’s this Linux system feature that allows thorough data examination. Regardless of particular input and output, data will be examined closely. Consequently, all lines that contain a definite character pattern are recognizable.
     This feature is called the I/O
 
What Is I/O Redirection?
 
The I/O redirection or the Input/Output redirection is a feature that simplifies a task. It also makes an assignment easier to accomplish. It can significantly increase your productivity especially if you are writing a complex program.
     Since input and output are distributed across streams, you should learn these three streams.
     The three streams:
 
Standard error – written as “stderr” and is assigned the number “2
Standard input – written as “stdin” and is assigned the number “0”
Standard output – written as “stdout” and is assigned the number “1”
 
     “stderr” is a stream that displays errors. These errors are likely related to a program’s unsuccessful execution. The terminal display is its default location.
     Input:
     #ls 98%
     Output:
     ls: Cannot access 98%: File or directory does not exist
     As the output explains, “98%” is a non-existent file or directory, and is, therefore, an error.
     “stdin” serves as data carrier from users to particular programs. Typically, “stdin” is received from devices (e.g. mouse, keyboard, and camera). To terminate an input, you should type in “ctrl –d”.
     Sample (for a program named “pen”):
     pen
     blue
     black
     ctrl –d
     As the sample shows, “5”, “blue”, and “black” are inputted to the “pen” program. There is no more additional input after “ctrl –d”.
     It follows that as you type in “5”, “blue”, and “black”, the “pen” program reports the input back to the terminal to check for standard errors.
     “stdout” is in charge of writing program-generated data. When the output redirection fails, the text is sent over to the terminal display.
     Sample output (for “mop”):
     mop sent to the terminal display via standard output
     As the sample shows, “mop” is sent to the terminal display via standard output. “mop” is then passed to the command line.
 
Over-writing & Appending
 
In Linux, each stream comes with a redirection command. A redirection command will over-write or append files. In the case of non-existent files, a new file will be written. 
     A redirection command will over-write a file if it comes with a single pointer or bracket. If it comes with double brackets, it will append a file.
     Redirection commands with a single bracket:
     2> - over-writes “stderr” files
     < - over-writes “stdin” files
     > - over-writes “stdout” files
     Redirection commands with double brackets:
     2>> - appends “stderr” files
     << - appends “stdin” files
     >> - appends “stdout” files
     For instance, a “write.txt” file under “mop” contains the components “1”, “2”, and “3”. A redirection command is then assigned.
     Sample # 1 (with over-written” file):
     mop > write.txt
     111
     112
     113
     As sample # 1 shows, “>” is used. Therefore, “1”, “2”, and “3” (as the original components of “write.txt” under “mop”) are over-written with “111”, “112”, and “113”.
     Sample # 2 (with appended file):
     mop >> write.txt
     111
     112
     113
     As sample # 2 shows, “>” is used. Therefore, “1”, “2”, and “3” (as the original components of “write.txt” under “mop”) are appended with “111”, “112”, and “113”.
 
The Role of Pipes & Filters
 
     In Linux, pipes are used for redirecting streams from a program to another program. When a program’s “stdout” is delivered through pipes, data will not be shown in the terminal display. Instead, only filtered data will be shown.
     A vertical bar or “|” is the representation of pipes.
     Sample:
     ls | so
     As the sample shows, the “ls” program uses a pipe to transfer data to the “so” program.
     Moreover, when using pipes, you should also learn about filters. Filters are standard commands that modify output and piped redirection.
     Five types of filters:
 
Grep – returns matching string pattern
Tee – supports the redirection of standard input to files and standard output
Wc – counts words, lines, and characters
Find – returns files with matching arguments
Tr – searches for string replacements
 
Special Redirection Commands & Operators
 
In Linux, special redirection commands exist. They are considered special since their redirection purpose is meant for a specific use.
     Special redirection commands and operators:
     XARGS
     XARGS redirects the output of one command then feeds it as the input to another command.
     Sample # 1:
     ls *.txt | lisp
     Sample # 2:
     ls *.txt | xargs lisp
     As sample # 1 shows, all files with the extension “.txt” are listed.
     As sample # 2 shows, XARGS is used. It means that the listed files are redirected to contain “lisp” in them.
     TEE
     TEE is a command that redirects file output and screen.
     Sample # 1:
     fdisk –l
     fdisk > fdskin.txt
     Sample # 2:
     fdisk –l | tee file7.txt
     As sample # 1 shows, two commands need execution to process the first command, then process the second command.
     In sample # 2, TEE was used. It means that instead of two commands, only one command needs to be executed. The file output and screen are already processed.
     - operator or hyphen operator
     The - operator redirects both output and input.
     Sample # 1:
     abc/temp/loc/file3.txt > /temp/loc/file4.txt
     Sample # 2:
     diff /temp/loc/file4.txt /temp/loc/file4.txt
     Sample # 3:
     abc/temp/loc/file3.txt | diff /temp/loc/file4.txt -
     As sample # 1 and sample # 2 shows, two commands need execution for both input and output.
     In sample # 3, since – operator is used, only one command is needed to redirect output and input.
 
Redirection Features for Advanced Uses
 
Due to the Linux redirection, advanced commands can be executed. The system can support even the most complicated features.
     Advanced uses for redirection:
     For thorough analysis of errors
     Sample:
     anne:~/newsoft> make all of the 2>&1 | less
     —output erased—
     For copying input
     Sample:
     Anne ~/test> date | tee file3 file4

     Anne ~/test> cat file3
     Thu Jun 10 09:10:15 CEST 2015

     Anne ~/test> cat file4
     Thu Jun 10 09:10:15 CEST 2015
     09:10:15 CEST 2015 up 22 days, 21:22, 15 users, load average: 0.04, 0.13, 0.26
     From separating output from error
     Sample:
     Anne:~/newsoft> make all of 2> /dev/pts/9
